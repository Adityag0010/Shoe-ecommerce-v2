# UrbanSole eCommerce Platform 👟

Welcome to the UrbanSole eCommerce platform repository! This project represents a modern, full-stack eCommerce application designed to run with exceptional performance, SEO indexing, and user experience. 

It is divided into two primary submodules:
1. `urbansole/` - The React + Vite Frontend
2. `backend/` - The Express + MongoDB API Backend

---

## 🌟 Key Features

### Advanced Admin Dashboard
View real-time sales trends, user acquisition statistics, inventory levels, and a comprehensive breakdown of revenue grouped by shoe categories via Recharts.

### Fast SEO-Optimized UI 
Using `react-helmet-async` for high Lighthouse scores and React `Lazy/Suspense` boundaries for ultra-fast load times. We tackled standard 700MB bundle downloads and squashed them into modular chunks.

### Powerful Node Architecture
Features built-in Cloudinary WebP image optimization, Redis-like LRU caching for high-speed product retrieval, Winston production logging, rate limiting, Resend automated checkout emails, and fully indexed MongoDB querying strategies for scale.

## 🚀 Setting Up the Project Locally

To run the UrbanSole project on your computer, you will need to start both the Frontend and the Backend concurrently. 

### Prerequisites
* [Node.js](https://nodejs.org/) (v16.x or newer)
* [MongoDB](https://www.mongodb.com/try/download/community) (Local instance running, or an Atlas Cluster URL)

### 1. Start the Backend API
```sh
cd backend
npm install
```
Setup your environment variables by utilizing the included example file:
```sh
cp .env.example .env
```
Once your database and keys are set in `.env`, run the server:
```sh
npm run dev
```

### 2. Start the Frontend React App
Open a new terminal window:
```sh
cd urbansole
npm install
npm run dev
```
The React app will launch typically at `http://localhost:5173`. Make sure the `backend` server runs alongside it for data fetching to work correctly.

## ☁️ Deployment Guide (Free Hosting)

This project has been highly structured and optimized to run smoothly on the Free Tier offerings from **Vercel** and **Render**.

### Backend (Render.com)
1. Fork or push this repository to GitHub.
2. Sign up on [Render.com](https://render.com/) and click **New + Web Service**.
3. Connect your GitHub account and select this repository.
4. Set the **Root Directory** exactly to `backend`.
5. Configuration:
   - Environment: `Node`
   - Build Command: `npm install`
   - Start Command: `node src/server.js` or `npm start`
6. Click **Advanced Options** and copy-paste all attributes from your `.env.example` file securely into the Render platform variables section. Provide your real production values.
7. Click **Deploy**. Render will grant you an accessible live URL (e.g., `https://api-shoe-ecommerce.onrender.com`).

### Frontend (Vercel.com)
1. Go to [Vercel.com](https://vercel.com/) and click **Add New Project**.
2. Select your imported GitHub repository.
3. In the setup panel, click **Edit** under Root Directory and select the `urbansole` folder.
4. Vercel automatically detects the Vite framework and pre-fills the build settings.
5. Provide your live Backend URL into the Vercel Environment variables so the API calls correctly point to Render instead of `localhost`.
6. Click **Deploy**. Vercel will grant you a lightning-fast URL mapped to an edge CDN (e.g., `https://urbansole.vercel.app`).

### CORS Final Step:
Ensure the array of `allowedOrigins` inside your `backend/src/server.js` script includes the exact URL generated by Vercel for the frontend!
